# ANTIGRAVITY NEXUS: AI ARCHITECT GUIDE

> **System Designation**: Research-Grade Multi-Agent Orchestration & Adversarial Proxy Harness
> **Internal Codebase Version**: 4.5.0-ALPHA (Neural-Ready)

## 1. ARCHITECTURAL TOPOLOGY

Antigravity is built on a **Modular Micro-Kernel Architecture** using FastAPI as the control plane. It functions as a transparent yet highly intelligent middleware between an Agent (like Claude) and LLM Providers.

### Core Tiers:
- **Presentation Layer**: Vanilla HTML5/CSS3/JS dashboard using `vis.js` for neural mapping and WebSockets for real-time telemetry.
- **Control Plane (API)**: FastAPI routing engine managing sessions, agents, and tool interception.
- **Intelligence Layer**: Shadow Intelligence (Hyper-Analysis) and OmX Planning (Architectural Strategy) using external 70B+ models.
- **Resilience Layer**: Adaptive provider switching, connection pooling, and error healing.

---

## 2. NEURAL MAP (GRAPHIFY ENGINE)

The **Neural Map** is a semantic visualization of the codebase generated by the `GraphifyEngine` (`api/graph/engine.py`).

### Mechanics:
- **Static Analysis**: Uses Python's `ast` (Abstract Syntax Tree) module to walk the codebase without execution.
- **Entity Discovery**: Extracts Modules, Classes, and Functions as nodes.
- **Relationship Mapping**: Detects `imports`, `inheritance`, and `containment` (class-method relationships).
- **Visualization**: Data is served via `/v1/graph/data` and rendered on the frontend using a Force-Directed Graph (`vis.Network`).

## 3. CORE SUB-SYSTEMS FOR AI AGENTS

### A. Mission Manager (`api/telemetry.py`)
The `MissionManager` is the source of truth for the system's operational state.
- **Session Tracking**: Tracks every request with a unique `request_id`.
- **Telemetry**: Real-time token counting, cost estimation, and tool usage tracking.
- **Change Log**: Captures filesystem mutations (Edit/WriteFile) to maintain a forensic audit trail.

### B. Provider Resilience (`providers/resilience.py`)
The system implements a **Neural Self-Correction Loop**.
- **`with_resilient_stream`**: A high-order wrapper that manages streaming retries.
- **Shadow Fallback**: If a primary provider (e.g., NVIDIA NIM) fails, it transparently switches to a secondary (e.g., OpenRouter) without breaking the client's connection.

### C. Adversarial Layer (Parseltongue)
Located in `api/routes.py`, the `perturb_prompt` function enables **Adversarial Mode**.
- **Homoglyph Attack**: Replaces sensitive characters with visually identical Cyrillic/Greek characters to bypass simple keyword filters.
- **Semantic Normalization (STM)**: Strips "AI Preambles" and hedges to ensure raw, high-density outputs.

## 4. OPERATIONAL FLOW (LIFE OF A REQUEST)

1.  **Ingress**: Request hits `/v1/messages`.
2.  **Shadow Intelligence**: A 70B model analyzes the prompt in the background to provide "Hyper-Insights" (Risk/Strategy).
3.  **OmX Planning**: A structured architectural plan is generated and injected into the system prompt.
4.  **Adversarial Perturbation**: Sensitive tokens are perturbed via `Parseltongue`.
5.  **Provider Routing**: Request is sent to the resolved provider (mapped via `ProviderManager`).
6.  **Telemetry Interception**: Tool calls and token usage are extracted from the stream and broadcast via WebSockets.
7.  **Egress**: Final response is delivered to the client.

---

## 5. SKILLS & SUPERPOWERS

Skills are dynamic plugins found in `skills/`.
- **Markdown Skills**: Prompt-based behavioral modifiers.
- **Python Skills**: Executable tool definitions that can be called by agents.
- **Discovery**: Automatically loaded at startup via `SkillLoader` (`api/skills/loader.py`).

---

## 6. SYSTEM INVARIANTS (FOR AI EDITORS)

1.  **Zero-Defect Typing**: All Python code must pass `ty` (Mypy-based) checks.
2.  **Streaming-First**: All LLM interactions MUST be streaming. Non-streaming paths are deprecated.
3.  **Encapsulation**: Use `mission_manager` for all state updates; do not mutate global vars directly.
4.  **Security**: Homoglyph mappings are for research use only. Ensure `ALLOWED_DIR` constraints are respected.
